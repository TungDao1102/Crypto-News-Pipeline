    Create `src/metrics.py` with:

    ```python
    """Daily metrics collector — in-memory counters, periodic file flush."""
    import asyncio
    import json
    import logging
    from collections import defaultdict
    from datetime import date
    from pathlib import Path
    from typing import Any

    logger = logging.getLogger(__name__)

    METRICS_DIR = Path("logs/metrics")
    FLUSH_INTERVAL = 300  # 5 minutes
    ```

    **DailyMetrics** class:
    - `__init__(self, metrics_dir: Path = METRICS_DIR)` — create metrics_dir (parents=True, exist_ok=True), initialize `self._counters: dict[str, int] = defaultdict(int)`, `self._latencies: list[float] = []`.
    - `increment(self, key: str) -> None` — `self._counters[key] += 1`.
    - `record_latency(self, seconds: float) -> None` — `self._latencies.append(seconds)`.
    - `_calculate_p95(self) -> float | None` — if no latencies, return None. Sort, compute index `int(len(sorted) * 0.95)`. Use local method so it's self-contained.
    - `approve_reject_ratio(self) -> float | None` — compute `approved / (approved + rejected)` from counters. Return None if total is 0.
    - `flush(self) -> None`:
      1. Build `metrics` dict with `date: date.today().isoformat()`, `counters: dict(self._counters)`.
      2. If latencies exist, compute P95 and include as `api_latency_p95`.
      3. File path: `self.metrics_dir / f"{date.today().isoformat()}.json"`.
      4. **Merge-on-write**: If file already exists, read existing JSON, merge counters by addition (new value += existing value), append latencies list, keep existing P95 if no new latencies (per D-16 security: merge rather than overwrite prevents tampering via file overwrite).
      5. Write with `json.dump(metrics, f, indent=2, ensure_ascii=False)`.
      6. Log `"Daily metrics flushed to %s"` with filepath.
    - `periodic_flush(self, interval: int = FLUSH_INTERVAL) -> None` — async generator: `while True: await asyncio.sleep(interval); self.flush()`.
