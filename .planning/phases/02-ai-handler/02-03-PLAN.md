**Action:** Implement `async def _process_message(self, msg: RawMessage) -> DraftContent | None`:

### 4. Wire processing loop in AIConsumer workers

Modify the worker loop created in Plan 02.

**Action:** In the worker task:
```python
async def _worker(self, worker_id: int):
    while True:
        qsize = self.raw_queue.qsize()
        if qsize > 10:
            logger.warning(f"Backpressure: raw_queue size={qsize}, processing rate may be insufficient")
        msg = await self.raw_queue.get()
        try:
            await self.rate_limiter.acquire()
            draft = await self._process_message(msg)
            if draft:
                await self.result_queue.put(draft)
        except Exception:
            logger.exception(f"Worker {worker_id}: unexpected error processing msg {msg.message_id}")
        finally:
            self.raw_queue.task_done()
```

**Acceptance criteria (D-13):**
- When raw_queue exceeds 10 messages, a WARNING log is emitted
- No pause/throttle mechanism is applied â€” per D-13 decision
- Workers loop until shutdown signal
- Each worker calls `rate_limiter.acquire()` before processing
- Successful drafts go to `result_queue`
- Unexpected errors don't crash the worker (caught and logged)
