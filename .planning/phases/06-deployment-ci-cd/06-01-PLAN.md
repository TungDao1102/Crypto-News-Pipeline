<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker build context → Docker image | Untrusted build context could inject malicious files into the image |
| Container runtime → host filesystem | Mounted volumes provide read/write access to host paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Docker image layers | mitigate | .env and sources.json excluded by .dockerignore — not baked into any image layer. Mounted as runtime volumes only. |
| T-06-02 | Tampering | Base image (python:3.14-slim) | mitigate | Pin rolling tag `python:3.14-slim`. Docker official image with automatic security scanning. |
| T-06-03 | Tampering | pip dependencies | mitigate | Install from requirements.txt with `--no-cache-dir`. Dependency ranges already pinned (e.g. telethon>=1.43,<2). |
| T-06-04 | Elevation of Privilege | Container process | mitigate | Do NOT mount /var/run/docker.sock. Container runs Python process only — no Docker-in-Docker. |
</threat_model>
