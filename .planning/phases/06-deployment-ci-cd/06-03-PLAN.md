    # Deployment Guide — Crypto News Pipeline

    (Brief intro: what this doc covers — from bare VPS to running pipeline)

    ## Prerequisites
    - Linux VPS (Ubuntu 22.04+ or Debian 12+ recommended)
    - Root or sudo access
    - Git installed on the VPS (for initial clone)
    - Docker Hub account (free tier)

    ## VPS Initial Setup
    Step-by-step shell commands (as fenced code blocks) for:
    1. Updating apt and installing Docker Engine (Docker official apt repo: https://docs.docker.com/engine/install/ubuntu/)
    2. Installing docker-compose plugin: apt install docker-compose-plugin
    3. Adding deploy user to docker group: sudo usermod -aG docker $USER
    4. Verifying: docker --version && docker compose version
    5. Cloning the repo: git clone https://github.com/<YOUR_ORG>/crypto-news-pipeline.git /opt/crypto-news-pipeline

    ## Directory Structure
    Show the expected layout:
    ```
    /opt/crypto-news-pipeline/
    ├── docker-compose.yml
    ├── data/
    │   ├── .env          (create manually)
    │   ├── sources.json  (create manually)
    │   ├── logs/         (auto-created by entrypoint)
    │   └── session/      (auto-created by entrypoint)
    └── ...
    ```

    ## Files You Need to Create (Manual Steps)
    ### .env
    - Copy .env.example content to /opt/crypto-news-pipeline/data/.env
    - Fill in all 8 required values (reference src/config.py REQUIRED_KEYS)
    - Recommend: nano /opt/crypto-news-pipeline/data/.env
    - Set file permissions: chmod 600 data/.env

    ### sources.json
    - Create based on your Telegram channel configuration
    - Reference the existing sources.default.json in the repo as a template

    ### telegram.session (after first run)
    - Note: this file is auto-created by Telethon on first auth
    - It will be persisted in the data/session/ volume

    ## GitHub Secrets Configuration
    Table with 6 secrets. For each secret, document:
    1. Secret name (as used in deploy.yml)
    2. What it is
    3. How to obtain it
    - DOCKERHUB_USERNAME: Your Docker Hub username
    - DOCKERHUB_TOKEN: Docker Hub PAT (Settings → Personal Access Tokens → New Token, scope: Read, Write, Delete)
    - SSH_HOST: VPS IP address or hostname
    - SSH_USER: SSH username on VPS (e.g., ubuntu, deploy)
    - SSH_PRIVATE_KEY: SSH private key (generate with ssh-keygen, add public key to ~/.ssh/authorized_keys on VPS)
    - SSH_KNOWN_HOSTS: Optional. Output of `ssh-keyscan <SSH_HOST>`. Strongly recommended for security.

    ## First Deploy
    1. Ensure VPS initial setup (Docker + repo clone) is done
    2. Create .env and sources.json in the data/ directory
    3. Configure all 6 GitHub Secrets in repo Settings → Secrets and variables → Actions
    4. Push a version tag: git tag v1.0.0 && git push origin v1.0.0
    5. Monitor the CI/CD pipeline in GitHub Actions tab
    6. After pipeline completes, SSH into VPS and verify

    ## Verification
    - docker ps (check container status is "Up" or "Healthy")
    - docker compose logs --tail 50 (check startup logs)
    - docker compose logs | grep "Entrypoint validation passed" (verify entrypoint ran)

    ## Troubleshooting
    ### Container Exits Immediately (Exit Code 1)
    Cause: Missing .env keys or placeholder values. ConfigError in src/config.py.
    Fix: Check docker compose logs for ConfigError message. Ensure all 8 env vars are filled.

    ### Telegram Session Lost (Re-auth on Every Deploy)
    Cause: telegram.session not persisted across container restarts.
    Fix: Ensure data/session/ volume is mounted. The volume path in docker-compose.yml is ./data/session:/app/session. Check that the directory exists on the host.

    ### SSH Host Key Verification Failed
    Cause: Ephemeral GitHub Actions runner has no known_hosts entry.
    Fix: Add SSH_KNOWN_HOSTS secret or set StrictHostKeyChecking=no in appleboy/ssh-action config.

    ### Health Check Shows Unhealthy
    Cause: pgrep health check may fail if process name doesn't match.
    Fix: Verify the container is running (docker ps). Health check is informational only — restart: unless-stopped handles crashes.

    ## Recovering Telegram Session
    If the session file is lost or corrupt:
    1. Stop the container: docker compose down
    2. Remove the old session file: rm data/session/telegram.session
    3. Start the container with a TTY to interact: docker compose run --rm pipeline
    4. The pipeline will prompt for phone number and OTP code
    5. After auth succeeds, verify session file exists: ls -la data/session/
    6. Restart normally: docker compose up -d
    (This section documents the reality that Telethon auth is interactive and cannot be fully automated.)

    ## Updating
    To deploy a new version:
    1. git tag v1.1.0 && git push origin v1.1.0
    2. Wait for CI/CD pipeline to complete
    3. The VPS automatically pulls and restarts via the workflow

    ## Rolling Back
    To roll back to a previous version:
    1. SSH into VPS
    2. docker compose down
    3. Manually edit docker-compose.yml to pin the previous image tag
    4. docker compose up -d
