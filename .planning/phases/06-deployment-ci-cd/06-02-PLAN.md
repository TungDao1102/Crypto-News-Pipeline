<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → Docker Hub | Runner authenticates with PAT to push images |
| GitHub Actions runner → VPS | Runner authenticates with SSH private key to execute remote commands |
| VPS Docker socket → container | Docker compose controls container lifecycle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Information Disclosure | GitHub Actions logs (SSH key) | mitigate | appleboy/ssh-action masks secret inputs in logs. `script_stop: true` fails fast on SSH errors. |
| T-06-06 | Information Disclosure | GitHub Actions logs (Docker Hub PAT) | mitigate | docker/login-action masks password input. PAT has push-only scope. |
| T-06-07 | Spoofing | Docker Hub image push | mitigate | Docker Hub PAT with limited scope (no admin). Secrets stored in GitHub encrypted secrets. |
| T-06-08 | Spoofing | SSH access to VPS | mitigate | SSH key-based auth (no passwords). key stored in GitHub Secrets. Private key should be passphrase-less deploy key with restricted permissions. |
| T-06-09 | Repudiation | Container health check failure | accept | pgrep health check is best-effort. If process hangs without crashing, restart policy handles it. App-level HealthCollector (Phase 5) provides deeper monitoring. |
| T-06-SC | Tampering | GitHub Actions ecosystem | mitigate | All actions pinned to major versions (@v4, @v7, @v1) — use explicit SHA pinning for production if needed. |
</threat_model>
